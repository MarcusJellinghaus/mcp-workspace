Add Tier 2 auxiliary checks: agent_reachable, allowed_signers, verify_head

Step 6 of issue #181: extend `verify_git` with three warning-tier
auxiliary checks. All three are gated on `signing_intent_detected`
and respect `signing_format_resolved` from Step 4. None of them
affects `overall_ok` (warning severity only).

- `agent_reachable` (openpgp / x509 only): resolves
  `gpg-connect-agent` via `shutil.which` and invokes it with `/bye`
  through the `_run` chokepoint with a 5-second timeout. Returncode
  zero → `value="reachable"`, ok=True. Non-zero → `value="unreachable"`
  with stderr (or stdout fallback) captured and truncated. Binary
  absent from PATH → `value="not found"` with an install hint. Key
  is absent from the result for ssh.

- `allowed_signers` (ssh only): extends the existing Tier 2
  `safe_repo_context` block with a fourth `_get_config` read for
  `gpg.ssh.allowedSignersFile` (the last config read in the block).
  Unset → `value="not configured"` with install hint; configured but
  path is not an existing file → `value="file missing: <path>"`;
  configured and present → `value=<path>`, ok=True. Key is absent
  from the result for openpgp / x509.

- `verify_head` (all formats): opens its own fresh
  `safe_repo_context` (logically separate from the config-read
  pipeline). Skips silently when `repo.head.is_valid()` is False
  (no commits yet). Invokes `repo.git.verify_commit("HEAD")` via
  GitPython. Success → `value="HEAD signature valid"`. A
  `GitCommandError` whose stderr contains `"no signature"` or
  `"not signed"` is treated as an opportunistic skip (HEAD is
  unsigned, key absent from result). Any other `GitCommandError`
  or unexpected exception during the repo open produces
  `severity="warning"` — never escalated to `"error"` per
  Decision #8.

Tests cover: ok / unreachable / binary-missing / format gating for
agent_reachable; ok / unset / file-missing / format gating for
allowed_signers; signed HEAD / no commits / two unsigned-substring
variants / unrelated GitCommandError / generic repo-open exception
for verify_head; plus an acceptance test confirming a failing
auxiliary does not flip `overall_ok` to False.
