Add Tier 2 config-only checks: signing_format, signing_key

Step 4 of issue #181: extend `verify_git` with the two config-only Tier 2
checks. Both keys are gated on `signing_intent_detected` from Step 3 —
when no signing flag is enabled, neither key is present in the result
(Decision #3).

- `signing_format`: reads `gpg.format`. Unset reports
  `"openpgp (default)"`, ok=True; explicit `openpgp` / `ssh` / `x509`
  echo back as the value with ok=True; any other value is an error
  (`"unknown: <raw>"`, severity error). The resolved format is captured
  into a function-local `signing_format_resolved` for downstream
  Steps 5 / 7.
- `signing_key`: reads `user.signingkey`. Set → ok=True,
  value="configured" (the key id is deliberately never echoed into the
  result or logged — Decision #11). Unset → ok=False with severity per
  Decision #10: error iff `commit.gpgsign` is true, otherwise warning,
  so a repo with only `tag.gpgsign` / `rebase.gpgSign` / `push.gpgSign`
  enabled and no key keeps `overall_ok` true.

Tests cover: keys absent without intent; all four format branches
including the unknown → error case; four severity branches for missing
key (commit-vs-tag/rebase/push); key-present path; the secret-not-
leaked guarantee against `CheckResult.value` and debug logs; and the
acceptance criterion that only-tag.gpgsign + missing key keeps
overall_ok=True. The existing Step 3 overall_ok-not-flipped-by-warning
test is updated to provide a signing key so it isolates Step 3's
behaviour.
