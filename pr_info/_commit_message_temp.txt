Add Tier 1 signing detection to verify_git

Step 3 of issue #181: implement the two Tier 1 signing-detection checks
in `verify_git` — `signing_intent` and `signing_consistency`. Both are
warning-severity, so failures do not flip `overall_ok`.

- `signing_intent`: OR over `commit.gpgsign`, `tag.gpgsign`,
  `rebase.gpgSign`, and `push.gpgSign`, all read through `_get_config`
  with `--type=bool` so git's truthy aliases (`yes`/`on`/`1`/`true`) are
  recognised uniformly. Reports `"not configured"` when no flag is
  enabled (with an `install_hint`), or `"detected: <comma-separated
  flags>"` otherwise.
- `signing_consistency`: when `commit.gpgsign=true`, warn if either
  `rebase.gpgSign` or `tag.gpgsign` is unset. Single key with
  concatenated `value` and `error` fields per Decision #12. Reports
  `"not applicable"` when commit signing is off.

When `git_repo` is not OK, both checks emit `ok=False`, severity
`warning`, `error="repository not accessible"` so callers see the
unknown state without overall_ok flipping.

Introduces a function-local `signing_intent_detected` boolean to be
consumed by later steps that gate Tier 2 checks. The variable is
debug-logged so tooling sees it as used in this step.

Tests extend the `_patch_baseline_ok` helper with four new signing-flag
parameters and add five new test classes covering: not-configured, each
detected variant, truthy-alias recognition through `--type=bool`,
single-key consistency shape (no `_rebase`/`_tag` split), the
no-repo-accessible warning path, and confirmation that warning failures
keep `overall_ok` true.
