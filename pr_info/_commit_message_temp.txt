Add Tier 1 baseline checks to verify_git

Step 2 of issue #181: implement the three baseline checks in
`verify_git` — `git_binary`, `git_repo`, and `user_identity`. Each check
populates its own `CheckResult` independently: a failure in one does not
prevent the others from running. `overall_ok` is now computed as the
conjunction of every error-severity check.

- `git_binary`: `shutil.which("git")` then `git --version` via `_run`
  with a 5s timeout; fails with `install_hint` when git is missing or
  non-runnable. The bare string "git" is reported in the value field;
  the resolved path is debug-logged only.
- `git_repo`: delegates to `repository_status.is_git_repository`.
- `user_identity`: reads `user.name` and `user.email` through
  `safe_repo_context` + `_get_config`; emits the missing key list and
  an `install_hint` when either is absent. Skipped with
  `error="repository not accessible"` when `git_repo` failed.

Tests cover the happy path, each independent failure mode (git not on
PATH, git non-runnable, missing repo, missing name/email, repo skip),
and the `overall_ok` recomputation. A new `_patch_baseline_ok` helper
mirrors the `verify_github` test pattern, dispatching to `shutil.which`,
`_run`, `is_git_repository`, `safe_repo_context`, and `_get_config`.
