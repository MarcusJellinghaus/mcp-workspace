verification: fix circular import with _permission_probes

Move `class CheckResult` above the `from ... import run_permission_probes`
statement in `github_operations/verification.py`. Previously the import
appeared earlier in source order than the class definition, so loading
`_permission_probes` returned a partially-initialized `verification`
module that did not yet have `CheckResult` bound, raising

    ImportError: cannot import name 'CheckResult' from partially
    initialized module 'mcp_workspace.github_operations.verification'

This broke nearly every test path that touched the package (including
unrelated `mcp_workspace.server` patches in test_reference_projects.py
and test_ssl.py, which fail with `AttributeError: module
'mcp_workspace' has no attribute 'server'` once `github_operations`
fails to load).

With `CheckResult` defined before the probes import, the partial-module
attribute lookup succeeds and the cycle resolves cleanly. The misleading
"imported after CheckResult is defined" comment is updated to accurately
describe the placement.
