Fix circular import between verification and _permission_probes

The `from ._permission_probes import run_permission_probes` statement in
verification.py was placed before `CheckResult` was defined, contradicting
the header comment that promised the import was deferred to break the
circular import. When `_permission_probes` tried to import `CheckResult`
back, the name did not yet exist on the partially initialized module and
mcp-coder crashed at startup, taking down every CI job that invokes it
(file-size, unit-tests, git-integration-tests, github-integration-tests).

Move the deferred import to after `class CheckResult(TypedDict)` and the
`logger` assignment so the design documented in the header comment is
actually realized. No behavior change.
