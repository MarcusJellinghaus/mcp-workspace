verification: fix circular import with _permission_probes

Move the `_permission_probes` import below the `CheckResult` TypedDict
definition in `github_operations/verification.py`. The accompanying
comment already declared this intent ("Imported after CheckResult is
defined to break the circular import"), but the actual import statement
was placed above the class definition.

`_permission_probes` imports `CheckResult` from `verification`, so when
`verification` is loaded first it would re-enter `verification` during
its own import phase before `CheckResult` was bound, raising:

    ImportError: cannot import name 'CheckResult' from partially
    initialized module 'mcp_workspace.github_operations.verification'

This broke every CI job that imports `mcp_coder` (file-size,
git-integration-tests, unit-tests, github-integration-tests). With
`CheckResult` now defined before the late import, the cycle resolves
cleanly. The pre-existing `# noqa: E402` and pylint
`wrong-import-position` suppressions already cover the placement.
